3-3-numbersProblem

Here are two interesting occurances when working with python

When the answer is a float when working with division,
you get the remainder as a decimal


x = 7 / 3
print(x)

Result

2.3333333333333335

But, you can round division with two //

x = 7 // 3 print(x)

Result

2

Problem with 0

A problem occurs when using floats getting an answer of 0

You would suspect 0, and a type of int, but this is not what happens

x = .1 + .1 + .1 - .3
print(x)
print(type(x))

Result

5.551115123125783e-17
<class 'float'>

This becomes problematic when working with money

we don't want to use floating point when calculating money

To solve this problem import the module decimal

from decimal import *

This states from the decimal module import everything

So the code will look like this at produce the correct result

from decimal import *

a = Decimal('.10')
b = Decimal('.30')
x = a + a + a - b print(x)
print(type(x))

Result

0.00
<class 'decimal.Decimal'>
